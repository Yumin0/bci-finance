## 一、三張單據的關係（與筑今相同）

```
資金分配申請單（母單，一張）
      │  核准後，可分批開出多張付款憑單，逐筆把額度用掉
      ▼
付款憑單（子單，可多張）
      │  勾了「預支」、且財務按過「確認付款」（轉「已付款」）後，
      │  才能從該張憑單內頁建立沖銷憑單
      ▼
暫付款沖銷憑單（孫單，一張付款憑單最多一張「活的」）
```

- 母單核准後得到一個「核准金額」＝可用額度。
- 底下的付款憑單一張一張把額度用掉，母單的「剩餘金額」隨之減少；剩餘金額用完時，母單自動結案（轉「已付款」）。
- 付款憑單勾了「**預支**」的，代表先把錢領出去、事後才核銷；財務按「確認付款」轉「已付款」後，該張憑單內頁才會出現「建立暫付款沖銷憑單」按鈕（**預支＋已付款兩條件缺一不可**——錢還沒實際付出去，就沒有「沖銷」可言）。
- **一張付款憑單只能建一張沖銷憑單**：沖銷單核准即代表這筆預支結案；被退回（不核准）的沖銷單不佔名額，可以重建。（筑今允許一張預支分多次沖銷，新系統簡化為一次沖完。）

## 二、單號規則

**送單日固定每週三**，方便財務一眼辨識是哪一週的單。單號都由系統自動編，不用自己想。

- 資金分配申請日期**只能選週三**，限未來三週內的某一個週三；系統會自動預選最近的送單週三（例：今天 7/13 週一 → 預設 7/15）。
- **資金分配單號 ＝ 申請日期 8 碼 ＋ 3 碼流水序**，例：20260715001、20260715002。
- **付款憑單採購單號 ＝ 母單號 11 碼 ＋ 3 碼流水序**，例：母單 20260715001 底下的憑單依序是 20260715001001、20260715001002。從採購單號就看得出這張憑單屬於哪張母單。
- **暫付款沖銷單號 ＝ 母付款憑單採購單號 14 碼 ＋ 3 碼流水序**，例：母憑單 20260415012001 → 沖銷單 20260415012001001。
- **日期原則**：資金分配的申請日期＝送單週三；**付款憑單與沖銷憑單的日期＝實際建單日**（付款憑單自動填當天不可改；沖銷憑單預設當天、可改）。各列表的週次篩選就依這些日期分週。

## 三、資金分配申請列表

> 這張列表出現在三個地方，欄位完全一致：首頁「我的申請紀錄」、「我的資金分配申請」、「全部申請紀錄」。使用者可用右上角「欄位」按鈕自行開關要看的欄位（會記住個人偏好）。「狀態」「單號」永遠顯示。

| 欄位 | 這欄在講什麼 | 邏輯 / 跨頁關聯 / 用途 |
|---|---|---|
| **狀態** | 這張單走到哪一關：草稿、審核中（會一併顯示目前卡在哪個審核步驟名稱）、已核准、不核准、已付款。 | 顏色與名稱可由「狀態標籤設定」自訂。 |
| **單號** | 這張申請單的正式編號。送出後才有；還是草稿時，這格顯示「繼續編輯」讓人點回去改。 | 全系統辨識這張單、付款憑單掛靠的依據。 |
| **申請處別** | 申請人所屬的「處」。 | 決定要送哪個處的主管審。新增申請時會依申請人所屬單位自動帶入。 |
| **申請課別** | 申請人所屬的「課」。 | 決定課級審核由誰簽。 |
| **申請人** | 送單的人。 | 追責與通知。 |
| **職務** | 顯示「單位＋職稱」，例：新技術研發部 D7-21B 課長。 | 表明申請人身分。 |
| **申請金額** | 申請人一開始填的金額。 | 送審的原始請款數字。 |
| **核准金額** | 審核後真正核准的金額，可能被砍。還沒核准顯示「-」。 | **是後續付款憑單的可用額度上限**。多關審核時，下一關會自動帶入上一關核准的金額當預設（例：課長核 9000，處長預設就帶 9000）；審核者**只能等於或下修，不能往上調高**。 |
| **剩餘金額** | 這張母單還剩多少額度沒用掉。即時算出來的，不是固定存的數字。還沒核准顯示「-」。 | 算法見下方「剩餘金額怎麼算」。歸零時母單自動結案。 |
| **出款帳戶** | 這筆錢從哪個帳戶付。 | 會帶到付款憑單。 |
| **費用項目** | 這筆花費的費用科目（例：2.1 行銷宣傳費用）。 | 會帶到付款憑單，供歸帳。 |
| **項目** | 這筆申請的白話說明（例：剩餘金額邏輯測試）。 | 讓人看懂用途；會帶到付款憑單。 |

### 剩餘金額怎麼算

```
剩餘金額 ＝ 核准金額 − 底下每一張付款憑單「目前佔用的金額」加總
```

每張付款憑單「佔用多少」，看它自己走到哪一關：

| 付款憑單狀態 | 佔用金額算法 | 為什麼 |
|---|---|---|
| 草稿 / 審核中 | 用「這張憑單填的金額」 | 一開單就先把額度預扣起來，避免同一筆錢被重複開單用掉。 |
| 已核准 / 已付款 | 用「這張憑單的核准金額」 | 核准階段金額可能被調整，改用核准後的數字。 |
| 不核准 | 算 0 | 這張被退掉，佔用的額度全數退回母單。 |

- 母單還沒核准（沒有核准金額）時，剩餘金額顯示「-」。
- 當財務對某張付款憑單按「確認付款」、且該母單剩餘金額歸零時，母單自動轉為「已付款」結案。剩餘沒用完就一直停在「已核准」，不會提早結案。

> 完整規則見同資料夾《資金分配_剩餘金額計算邏輯規格》。

| **採購單號** | 這張憑單的正式編號＝母單號＋3 碼流水（見第二節）。 | 單號本身就是連結，點了進詳細頁（審核管理則進審核頁）。 |

| **付款對象** | 這筆錢付給誰（受款人）。 | 開單時搜尋全部付款對象選定。 |
| **付款方式** | 怎麼付（銀行轉帳、臨櫃匯款、現金、國外跨境匯款）。 | 財務作業依據。 |
| **核准金額** | 這張憑單經審核核准的金額。還沒核准顯示「-」。 | 審核人可在審核頁填寫或下修（承接上一步、上限＝母單剩餘額度＋本張佔用）。 |
| **實際付款金額** | 財務實際撥出去的錢。**只有「已付款」的憑單才顯示**（＝核准金額），其餘顯示「-」。 | 財務按「確認付款」後成立。 |
| **發票憑證** | 這張憑單夾帶的附件檔（報價單／合約書／發票）。 | 顯示上傳時的原始檔名、一檔一行，點檔名可預覽。 |

### 付款憑單的金額是怎麼組成的（重要）

- 付款明細每一組有三個數字欄：**未稅金額、稅額、總額**；未稅金額與稅額可留 0、直接填總額。
- 這張憑單的「金額」＝**各組總額加總**。
- 審核者審這張憑單時可填寫或**下修**核准金額（不能調高；上限受母單剩餘額度把關），與資金分配申請單一致。

> 財務的「付款憑單管理」頁欄位與上表相同，尾端另多兩欄：**付款分類**（財務審核時選的出帳分類，見第十節）與**付款執行**（「確認付款」按鈕；已付款的顯示灰色「已付款」）。

## 六、與筑今系統的差異

新系統的欄位、單號與金額規則都以筑今為基準，日常操作邏輯一致。目前的差異只有這幾條：

| 項目 | 筑今 | 新系統 |
|---|---|---|
| 預支的沖銷 | 一張預支憑單可分多次沖銷，審核時勾「沖銷已結束」收尾 | **一張付款憑單只能建一張沖銷憑單**，沖銷單核准即結案，不需要勾選框 |
| 次帳戶 | 財務審核時另有「次帳戶」下拉 | **不做次帳戶**，只有付款分類（見第十節） |
| 付款憑單管理頁的批次確認付款 | 可勾選多筆一次執行 | 尚未提供，目前一筆一筆按「確認付款」 |
| 付款憑單管理頁的匯出報表 | 有 | 尚未提供 |

## 七、名詞小抄（避免混淆）

- **母單 / 資金分配申請單**：先講要花多少錢、核准後給一個可用額度。
- **子單 / 付款憑單**：實際付錢給某對象時開的單，一張母單可開多張。
- **孫單 / 暫付款沖銷憑單**：預支的錢實際付出去（已付款）後，用來把預支沖銷結案的單；新系統一張付款憑單最多一張「活的」沖銷單。
- **預支**：先把錢領出去、事後才核銷的付款憑單，開單時勾選「預支（需事後沖銷）」。
- **申請金額**：一開始填的。**核准金額**：審核後真正准的。**剩餘金額**：母單還沒被用掉的額度（即時算）。
- **佔用**：付款憑單一開出來，就先把母單的額度扣住（預扣），避免重複使用。
- **付款分類**：財務審核時加註「這筆錢怎麼出帳」的作業分類（整批匯、單獨匯、現金…），見第十節。

## 八、防呆與限制（規則總表）

> **這一節是什麼**：系統裡「跟金額與流程有關的防呆規則」全部集中在這張表。操作被系統擋下來時，可以來這裡查是撞到哪一條、為什麼。
>
> **兩道門觀念**：「畫面上擋」＝填表時立刻提醒、不給送出；「存檔時擋」＝資料真正寫入前再檢查一次，不合就拒收，是保證帳算得對的最後防線。理想是兩道門都有，下表逐條標明。

### 8-1 規則總表

| 規則 | 目前狀況 |
|---|---|
| 付款明細每一組的金額必須大於 0 才能送出（資金分配看「費用」、付款憑單看「總額」） | ✅ 送出時擋；**存草稿不檢查**（刻意設計：草稿允許先存一半） |
| 付款憑單金額必須大於 0 才能存檔（**草稿也擋**） | ✅ 兩道門。負數金額會讓母單剩餘不減反增，故草稿也不放行 |
| 同一張母單底下，所有付款憑單金額加總不能超過核准金額 | ✅ 兩道門。建立／修改憑單時重算剩餘額度，超過拒收並逐張點名佔用中的憑單 |
| 審核核准金額必須大於 0（要拒絕請按「不核准」，不是填 0 元核准） | ✅ 兩道門 |
| 審核核准金額只能等於或下修、不能往上調高（承接上限＝上一關核准金額，第一關＝申請金額） | ✅ 兩道門 |
| 母單核准金額下修時，不可低於底下憑單已佔用的總額 | ✅ 兩道門。擋下時逐張點名：哪天、誰（本人寫「你自己」）、哪張憑單（採購單號）、各佔多少，並寫明最低可填金額 |
| 付款憑單審核核准金額不可超過母單剩餘額度＋本張佔用 | ✅ 兩道門 |
| 付款憑單「總額」不可低於「未稅＋稅額」的合計 | ✅ 送出時擋。剛好等於或大於都放行；直接填總額、未稅稅額留 0 的用法（代墊匯還、國稅局稅額）不受影響 |
| 母單剩餘金額歸零（或更早已 ≤0）時，不顯示「建立付款憑單」按鈕 | ✅ 畫面擋（不用等轉「已付款」）；存檔時的超額檢查照舊保留當最後防線 |
| 財務「確認付款」只能對「已核准」的憑單按，不能重複按、不能跳關 | ✅ 存檔時擋 |
| 資金分配申請日期只能選週三、限未來三週內 | ✅ 有擋 |
| 暫付款沖銷憑單只能從「已付款＋預支」的付款憑單建立 | ✅ 兩道門 |
| **一張付款憑單只能建一張沖銷憑單**（草稿／審核中／已核准佔名額；被退回的不算、可重建） | ✅ 兩道門。擋下時點名既有沖銷單的單號與狀態 |
| 沖銷金額（付款明細各組總額加總）必須大於 0、且不可超過原預支憑單實際付款金額（＝該憑單的核准金額，因為只有「已付款」的憑單能建沖銷單，已付款時實付＝核准金額） | ✅ 存檔時擋。為什麼：沖銷是在核銷帳上掛的「暫付款」，天花板就是當初實際撥出去的錢；沖超過等於核銷不存在的預支，暫付款科目會變負數 |
| 兩人「同一秒」對同一張母單建憑單，理論上可雙雙過關而超額 | 📌 已知限制、暫不處理（以目前使用量發生機率極低，使用量變大再補強） |

### 8-2 提示訊息原則（所有防呆共同適用）

**每一條防呆擋下操作時，訊息都必須讓一般職員看得懂三件事：①發生了什麼事 ②為什麼被擋 ③接下來該怎麼做。** 不可以只寫「金額不合法」「驗證失敗」這種話。

- 訊息裡的人、單、日期都要**點名到具體**：幾月幾號、誰、哪張單（含單號）、金額多少；建單的人就是看訊息的人時，直接寫「你自己建立的」。
- **不可以用「承辦人」這種抽象稱呼**；「接下來該怎麼做」也要指名道姓（直接請那個人去刪除或調整該憑單）。
- **呈現方式：畫面中央彈窗（全站共用元件）**，按「我知道了」關閉，支援多行點名式長訊息。理由：長表單送出按鈕在頁面底部，錯誤只放頁頂紅字使用者看不到；原生瀏覽器跳窗也已全數換成同一個彈窗。

點名式訊息範例（下修核准金額被擋時）：

> 「這張資金分配單底下已經有付款憑單在用額度：
> ・7 月 10 日 王小明 建立的「Fireflies 6月平台費」付款憑單（採購單號 20260715001001），金額 3,000 元
> 合計已佔用 3,000 元。核准金額如果改成 2,000 元，會低於已經用掉的錢，帳會對不起來，所以無法儲存。
> 請填 3,000 元以上；或請 王小明 先刪除／調降上面那張憑單的金額，再回來下修。」

## 九、三張單據內頁欄位總表（資金分配 × 付款憑單 × 暫付款沖銷）

> **這一節是什麼**：前面幾節只講「列表」欄位；這一節把三張單「點進去之後的內頁（申請／建立表單）」的欄位全部攤開，一張總表看懂三件事：
> 1. 每張單內頁有哪些欄位
> 2. 哪些欄位**必填**
> 3. 哪些欄位的內容會**繼承**（自動帶入）到下一張單
>
> 欄位清單依實際表單設定與系統行為逐欄核對（2026-07-14 版）。管理員在「表單設定」頁調整過欄位的話，請回來同步這張表。

### 繼承的方向

```
資金分配申請單 ──(核准後建立)──▶ 付款憑單 ──(已付款＋預支才能建立)──▶ 暫付款沖銷憑單
```

- 付款憑單建立時，母單的基本資料整排帶過來（唯讀），頁面最上方另有母單完整摘要卡（單號／處別／課別／日期／申請人／職稱／幣別／機構／出款帳戶／費用項目／項目＋核准金額＋剩餘金額）。
- 暫付款沖銷憑單建立時，帶入付款憑單的處別／課別／申請人／職稱／採購單號（唯讀），**付款明細逐組帶入**付款憑單的摘要用途／未稅金額／稅額／總額（可修改、可增刪組），詳細頁顯示「關聯付款憑單」連結（顯示採購單號）。
- **每張單「決定金額」的欄位一律不帶入、自己填**：資金分配→付款憑單三個金額欄全不帶；付款憑單→沖銷憑單的「總額」（＝沖銷金額）自填。未稅金額／稅額／總額屬於沖銷時的預帶參考值（總額預帶＝最常見的全額沖銷情境），皆可修改。系統用「剩餘額度／原憑單實付金額」把關上限。

## 十、付款分類

> **這一節是什麼**：財務審核付款憑單／沖銷憑單時，可以加註「這筆錢要用哪種方式出帳」的分類（例：整批匯、單獨匯、匯款、銀付、現金、薪轉），方便財務安排出帳作業。

### 誰在哪裡選

- 付款憑單的審核走到**財務的審核關卡**（第三處支出課、第三處處長）時，審核畫面除了核准／不核准，還有一個「付款分類」下拉可以選（選填，不選也能核准）。課長、處長等一般主管關卡不會出現這個欄位。
- **後面的關卡會自動承接前面選的值**：支出課選了「整批匯」，到處長那關就預帶「整批匯」，處長可以改選別的——邏輯跟核准金額的逐關承接一樣，以最後一關選的為準。
- **暫付款沖銷憑單的審核也一樣有付款分類**：預設帶入原付款憑單審核時最後選定的付款分類，可以改。（沖銷憑單的審核流程要有財務關卡才會出現這個欄位；流程內容由管理員在審核流程設定調整。）

### 選完在哪裡看得到

- **審核管理的財務群組頁籤**（財務人員、第三處處長）：列表尾端有「付款分類」欄，顯示每張憑單目前最新選定的分類；還沒選的顯示「-」。
- **財務的「付款憑單管理」頁**：同樣有「付款分類」欄（見第四節）。
- 審核頁的審核進度裡，已審過的關卡旁邊也會顯示當時選的付款分類。

### 選項誰來管

- 選項清單在「**系統設定 → 財務自定義設定 → 支出欄位設定**」的「付款分類」區塊維護，財務可自行新增、改名、刪除，不用找工程師。
- 刪除某個選項不影響已經審核過的舊憑單——舊憑單上記的分類會照原樣保留顯示。

### 與筑今的差異

- 筑今在同一個畫面還有「次帳戶」下拉與「此付款單的暫付款沖銷已結束」勾選框；新系統**都不做**——次帳戶整個系統用不到，「沖銷已結束」則因為新系統一張付款憑單只能建一張沖銷憑單（核准即結案），不需要另外勾。
