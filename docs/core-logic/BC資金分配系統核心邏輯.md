## 一、兩張單據的關係（與筑今相同）

```
資金分配申請單（母單，一張）
      │  核准後，可分批開出多張付款憑單，逐筆把額度用掉
      ▼
付款憑單（子單，可多張）
```

- 母單核准後得到一個「核准金額」＝可用額度。
- 底下的付款憑單一張一張把額度用掉，母單的「剩餘金額」隨之減少。
- 剩餘金額用完時，母單自動結案（轉「已付款」）。

## 二、單號規則（沿用筑今基準）

**送單日固定每週三**，方便財務辨識是哪一週的單。

- 資金分配申請日期**只能選週三**，限未來三週內的某一個週三；系統依今天預設最近的送單週三（例：今天 7/13 週一 → 預設 7/15）。
- **資金分配單號 ＝ 申請日期 8 碼 ＋ 3 碼流水序**，例：`20260715001`、`20260715002`。（新系統實際單號即此格式，已對齊。）
- **付款憑單採購單號 ＝ 母單號 11 碼 ＋ 3 碼流水序**，例：母單 `20260715001` → `20260715001001`、`20260715001002`。
  - 📌 2026-07-14 修正：舊程式流水碼寫死 `001`，同一張母單底下開第二、三張憑單會拿到**一模一樣的採購單號**（撞號）。已改為依同母單既有憑單遞增，測試站與正式機的舊撞號單也依建單順序補正重編。
- **暫付款沖銷單號 ＝ 母付款憑單採購單號 14 碼 ＋ 3 碼流水序**（2026-07-14 Yumin 拍板新增），例：母憑單 `20260415012001` → 沖銷單 `20260415012001001`。建立沖銷憑單時自動編號。
- **日期原則**（2026-07-14 Yumin 拍板）：資金分配申請日期＝送單週三（維持不變）；**付款憑單與暫付款沖銷憑單的日期＝實際建單日**（付款憑單唯讀自動填；沖銷憑單預設當天、可改），讓每張單的日期反映實際建單時間。付款憑單審核管理的週次篩選隨之依實際建單週分組，不再跟著母單的送單週。

## 三、資金分配申請列表

> 這張列表出現在三個地方，欄位完全一致：首頁「我的申請紀錄」、「我的資金分配申請」、「全部申請紀錄」。使用者可用右上角「欄位」按鈕自行開關要看的欄位（會記住個人偏好）。「狀態」「單號」永遠顯示。

| 欄位 | 這欄在講什麼 | 邏輯 / 跨頁關聯 / 用途 |
|---|---|---|
| **狀態** | 這張單走到哪一關：草稿、審核中（會一併顯示目前卡在哪個審核步驟名稱）、已核准、不核准、已付款。 | 顏色與名稱可由「狀態標籤設定」自訂。 |
| **單號** | 這張申請單的正式編號。送出後才有；還是草稿時，這格顯示「繼續編輯」讓人點回去改。 | 全系統辨識這張單、付款憑單掛靠的依據。 |
| **申請處別** | 申請人所屬的「處」。 | 決定要送哪個處的主管審。新增申請時會依申請人所屬單位自動帶入。 |
| **申請課別** | 申請人所屬的「課」。 | 決定課級審核由誰簽。 |
| **申請人** | 送單的人。 | 追責與通知。 |
| **職務** | 顯示「單位＋職稱」，例：新技術研發部 D7-21B 課長。 | 表明申請人身分。 |
| **申請金額** | 申請人一開始填的金額。 | 送審的原始請款數字。 |
| **核准金額** | 審核後真正核准的金額，可能被砍。還沒核准顯示「-」。 | **是後續付款憑單的可用額度上限**。多關審核時，下一關會自動帶入上一關核准的金額當預設（例：課長核 9000，處長預設就帶 9000）；審核者**只能等於或下修，不能往上調高**。 |
| **剩餘金額** | 這張母單還剩多少額度沒用掉。即時算出來的，不是固定存的數字。還沒核准顯示「-」。 | 算法見下方「剩餘金額怎麼算」。歸零時母單自動結案。 |
| **出款帳戶** | 這筆錢從哪個帳戶付。 | 會帶到付款憑單。 |
| **費用項目** | 這筆花費的費用科目（例：2.1 行銷宣傳費用）。 | 會帶到付款憑單，供歸帳。 |
| **項目** | 這筆申請的白話說明（例：剩餘金額邏輯測試）。 | 讓人看懂用途；會帶到付款憑單。 |

### 剩餘金額怎麼算（新系統目前的實際邏輯）

```
剩餘金額 ＝ 核准金額 − 底下每一張付款憑單「目前佔用的金額」加總
```

每張付款憑單「佔用多少」，看它自己走到哪一關：

| 付款憑單狀態 | 佔用金額算法 | 為什麼 |
|---|---|---|
| 草稿 / 審核中 | 用「這張憑單填的金額」 | 一開單就先把額度預扣起來，避免同一筆錢被重複開單用掉。 |
| 已核准 / 已付款 | 用「這張憑單的核准金額」 | 核准階段金額可能被調整，改用核准後的數字。 |
| 不核准 | 算 0 | 這張被退掉，佔用的額度全數退回母單。 |

- 母單還沒核准（沒有核准金額）時，剩餘金額顯示「-」。
- 當財務對某張付款憑單按「確認付款」、且該母單剩餘金額歸零時，母單自動轉為「已付款」結案。剩餘沒用完就一直停在「已核准」，不會提早結案。

> 此算法與筑今基準一致，完整規則見同資料夾《資金分配_剩餘金額計算邏輯規格》。

### 付款憑單的金額是怎麼組成的（重要）

- 付款明細每一組有三個數字欄：**未稅金額、稅額、總額**；未稅金額與稅額可留 0、直接填總額。
- 這張憑單的「金額」＝**各組總額加總**。
- 審核者審這張憑單時可填寫或**下修**核准金額（不能調高；上限受母單剩餘額度把關），與資金分配申請單一致。

## 七、名詞小抄（避免混淆）

- **母單 / 資金分配申請單**：先講要花多少錢、核准後給一個可用額度。
- **子單 / 付款憑單**：實際付錢給某對象時開的單，一張母單可開多張。
- **申請金額**：一開始填的。**核准金額**：審核後真正准的。**剩餘金額**：母單還沒被用掉的額度（即時算）。
- **佔用**：付款憑單一開出來，就先把母單的額度扣住（預扣），避免重複使用。

## 八、防呆與限制（規則總表）

> **這一節是什麼**：系統裡「跟金額與流程有關的防呆規則」全部集中在這張表。2026-07-14 全面體檢盤點出的缺口，經 Yumin 逐項拍板後已全數補完（或明確記錄暫不處理），原缺口清單整併於此。
>
> **兩道門觀念**：「畫面上擋」＝填表時立刻提醒、不給送出，體驗好但可能被繞過；「存檔時擋」＝資料真正寫入前再檢查一次，不合就拒收，是保證帳算得對的最後防線。理想是兩道門都有，下表逐條標明。

### 8-1 規則與目前狀況

| 規則 | 目前狀況 |
|---|---|
| 付款明細每一組的金額必須大於 0 才能送出（資金分配看「費用」、付款憑單看「總額」） | ✅ 送出時擋；**存草稿不檢查**（刻意設計：草稿允許先存一半） |

| 審核核准金額必須大於 0（要拒絕請按「不核准」，不是填 0 元核准） | ✅ 兩道門 |

| 母單核准金額下修時，不可低於底下憑單已佔用的總額 | ✅ 兩道門。擋下時逐張點名：哪天、誰（本人寫「你自己」）、哪張憑單（採購單號）、各佔多少，並寫明最低可填金額 |
| 付款憑單審核核准金額不可超過母單剩餘額度＋本張佔用 | ✅ 兩道門 |

| 暫付款沖銷憑單只能從「已付款＋預支」的付款憑單建立 | ✅ 兩道門 |

### 8-2 提示訊息原則（所有防呆共同適用，Yumin 指定）

**每一條防呆擋下操作時，訊息都必須讓一般職員看得懂三件事：①發生了什麼事 ②為什麼被擋 ③接下來該怎麼做。** 不可以只寫「金額不合法」「驗證失敗」這種話。

- 訊息裡的人、單、日期都要**點名到具體**：幾月幾號、誰、哪張單（含單號）、金額多少；建單的人就是看訊息的人時，直接寫「你自己建立的」。
- **不可以用「承辦人」這種抽象稱呼**；「接下來該怎麼做」也要指名道姓（直接請那個人去刪除或調整該憑單）。
- **呈現方式：畫面中央彈窗（全站共用元件）**，按「我知道了」關閉，支援多行點名式長訊息。理由：長表單送出按鈕在頁面底部，錯誤只放頁頂紅字使用者看不到；原生瀏覽器跳窗也已全數換成同一個彈窗。

點名式訊息範例（下修核准金額被擋時）：

> 「這張資金分配單底下已經有付款憑單在用額度：

> 合計已佔用 3,000 元。核准金額如果改成 2,000 元，會低於已經用掉的錢，帳會對不起來，所以無法儲存。
> 請填 3,000 元以上；或請 王小明 先刪除／調降上面那張憑單的金額，再回來下修。」

## 九、三張單據內頁欄位總表（資金分配 × 付款憑單 × 暫付款沖銷）

> **這一節是什麼**：前面幾節只講「列表」欄位；這一節把三張單「點進去之後的內頁（申請／建立表單）」的欄位全部攤開，一張總表看懂三件事：
> 1. 每張單內頁有哪些欄位
> 2. 哪些欄位**必填**
> 3. 哪些欄位的內容會**繼承**（自動帶入）到下一張單
>
> 欄位清單依 2026-07-14 測試站的實際表單設定與程式行為逐欄核對，不是憑印象整理。表單欄位可在「表單設定」頁調整，調整後請回來同步這張表。

### 繼承的方向

```
資金分配申請單 ──(核准後建立)──▶ 付款憑單 ──(已付款＋預支才能建立)──▶ 暫付款沖銷憑單
```

- 付款憑單建立時，母單的基本資料整排帶過來（唯讀），頁面最上方另有母單完整摘要卡（單號／處別／課別／日期／申請人／職稱／幣別／機構／出款帳戶／費用項目／項目＋核准金額＋剩餘金額）。
- 暫付款沖銷憑單建立時，帶入付款憑單的處別／課別／申請人／職稱／採購單號（唯讀），**付款明細逐組帶入**付款憑單的摘要用途／未稅金額／稅額／總額（可修改、可增刪組），詳細頁顯示「關聯付款憑單」連結（顯示採購單號）。
- **每張單「決定金額」的欄位一律不帶入、自己填**：資金分配→付款憑單三個金額欄全不帶；付款憑單→沖銷憑單的「總額」（＝沖銷金額）自填。未稅金額／稅額／總額屬於沖銷時的預帶參考值（總額預帶＝最常見的全額沖銷情境），皆可修改。系統用「剩餘額度／原憑單實付金額」把關上限。
