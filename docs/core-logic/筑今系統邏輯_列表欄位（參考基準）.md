## 一、先看懂兩張單據的關係

筑今的錢是「兩層單據」在跑：

```
資金分配申請單（母單，一張）
      │  核准後，可以分批開出多張付款憑單
      ▼
付款憑單（子單，可多張）
```

- **資金分配申請單**：先講「這筆錢要花多少、花在哪」，送簽核，核准後得到一個「可用額度」。
- **付款憑單**：實際要付錢給某個對象時開的單，一張母單底下可以開很多張，逐筆把額度用掉。
- 兩張單各自都要跑審核（送出 → 各層審核 → 財務最終處理）。

理解這層母子關係，才看得懂為什麼「付款憑單」很多欄位是從「資金分配」延續下來的。

## 二、單號規則（重要，貫穿兩張單）

**送單日固定為每週三**，這是財務為了「一眼分辨是哪一週的單」訂的規矩。

### 資金分配申請單號
- 申請日期欄位**只能選週三**，而且限「未來三週內的某一個週三」。
- 系統會依今天預設最近的可送單週三。例：今天 7/13（週一）→ 預設申請日期 7/15（週三）。
- **單號 ＝ 申請日期（西元年月日 8 碼）＋ 3 碼流水序**。同一個週三送出的單，流水序 001、002、003… 往下遞增。
  - 例：7/15 當期 → `20260715001`、`20260715002`、`20260715003`…
- 單號自動生成，申請人不用自己編。財務看單號前 8 碼就知道是哪一週。

### 付款憑單「採購單號」
- 一張資金分配單可以轉成多張付款憑單。
- **採購單號 ＝ 母單資金分配單號（11 碼）＋ 3 碼流水序**。
  - 例：母單 `20260715001` → 底下付款憑單 `20260715001001`、`20260715001002`。
- 從採購單號就能直接看出這張付款憑單屬於哪張母單。

## 三、資金分配申請列表 — 逐欄說明

畫面欄位順序：狀態｜單號｜申請處別｜申請課別｜申請人｜職稱｜申請金額｜核准金額｜剩餘金額｜出款帳戶｜費用項目｜項目

| 欄位 | 這欄在講什麼 | 用途 / 跨頁關聯 |
|---|---|---|
| **狀態** | 這張單走到哪一關，通常是「狀態＋目前關卡」合併顯示，例：新單-課長、未核准-課級審核、已核准-財務長審核、已付款。 | 讓申請人一眼知道卡在哪、能不能繼續動作。 |
| **單號** | 這張申請單的正式編號（規則見第二節）。 | 全系統辨識這張單；付款憑單掛在這個單號底下。 |
| **申請處別** | 申請人所屬的「處」。 | 決定送哪個處的主管審核。 |
| **申請課別** | 申請人所屬的「課」。 | 決定課級審核由誰簽。 |
| **申請人** | 送這張單的人。 | 追責與通知對象。 |
| **職稱** | 申請人的職務。 | 顯示申請人身分。 |
| **申請金額** | 申請人一開始送出時填的金額（例：3000）。 | 送審的原始請款數字，也是第一關核准金額的預設值。 |
| **核准金額** | 審核後真正被核准的金額，可能被下修。 | **是後續付款憑單的可用額度上限**。此欄會「逐關承接、被複寫」，規則見下方。 |
| **剩餘金額** | 這張母單還有多少額度沒被用掉。即時計算，不是固定存的數字。 | ＝核准金額 − 底下付款憑單目前佔用的金額。詳見下方「剩餘金額怎麼算」。 |
| **出款帳戶** | 這筆錢從哪個帳戶付出去。 | 會一路帶到付款憑單。 |
| **費用項目** | 這筆花費的費用科目（例：6.2 數位平台費用）。 | 會帶入付款憑單，供財務歸帳。 |
| **項目** | 這筆申請的白話說明（例：Fireflies.ai 6月平台費(4人)）。 | 讓人看懂用途；會帶入付款憑單。 |

### 核准金額怎麼承接、複寫（已確認）
- 申請人送出時填「申請金額」（例 3000）。
- 每一輪審核都要勾「核准」或「不核准」。**勾核准時，核准金額預設帶入上一關的核准金額**（第一關預設＝申請金額），審核者可再自行下修。
- 逐關承接：第一關核 2500 → 送到第二關時金額就是 2500 → 一路到最後一關，**最後一關核准的數字＝最終核准金額**。
- 審核者**只能等於或下修**上一關的金額，**不能往上調高**。
- 所以「核准金額」是會被逐關複寫的欄位，列表顯示的是目前最新（走完後即最終）的核准金額。

### 剩餘金額怎麼算（依《資金分配_剩餘金額計算邏輯規格》）
剩餘金額不是固定存的數字，是每次有相關動作就即時重算：

```
剩餘金額 ＝ 母單核准金額 − 底下每張付款憑單「目前佔用的金額」加總
```

每張付款憑單佔用多少，看它走到哪一關：

| 付款憑單狀態 | 佔用金額 | 說明 |
|---|---|---|
| 新單／審核中 | 該憑單的申請金額（＝填的總額） | **一建立就先預扣**，避免同一筆錢被重複用掉 |
| 已核准 | 核准金額 | 核准階段金額可能被調整，改用核准值 |
| 已付款 | 實際付款金額（≤核准金額） | |
| 不核准 | 0 | 額度全數退回母單 |

> 完整公式、狀態機與結案條件見同資料夾《資金分配_剩餘金額計算邏輯規格 v6》。

## 四、付款憑單列表 — 逐欄說明

畫面欄位順序：狀態｜採購單號｜費用項目｜項目｜付款對象｜付款方式｜核准金額｜實際付款金額｜發票憑證

| 欄位 | 這欄在講什麼 | 用途 / 跨頁關聯 |
|---|---|---|
| **狀態** | 這張付款憑單走到哪一關（例：新單-課長、未核准-課級審核、已付款）。 | 看卡在哪關。 |
| **採購單號** | 這張付款憑單的編號（＝母單號＋3 碼流水，見第二節）。 | 看得出它屬於哪張資金分配母單。 |
| **費用項目** | 這筆付款的費用科目。 | **從母單資金分配帶過來**，不另外重選。 |
| **項目** | 這筆付款的白話說明。 | 從母單帶過來。 |
| **付款對象** | 錢實際付給誰（廠商、職員、國稅局等，例：國稅局_S110、黃鈺閔_E0023）。 | 付款憑單特有；同一張母單可拆給不同對象各開一張。 |
| **付款方式** | 怎麼付（銀行轉帳、臨櫃繳款、現金等）。 | 財務實際作業依據。 |
| **核准金額** | 這張付款憑單被核准的金額（＝申請者填的「總額」經審核核准後成立）。 | 受母單剩餘額度限制；也是「實際付款金額」的上限。**審核者只針對總額按核准／不核准，不編輯金額**（與資金分配可下修核准金額不同）。 |
| **實際付款金額** | 財務實際付出去、認列的金額。 | 見下方規則。 |
| **發票憑證** | 這筆付款夾帶的證明附件（報價單、合約書、發票等）。 | 以可點開的檔案連結呈現（例：檔案1 檔案2），供核銷佐證。 |

### 付款憑單的金額怎麼組成（未稅金額／稅額／總額）
- 付款憑單申請者填的數字欄位有三個：**未稅金額、稅額、總額**。
- 列表上的「核准金額／實際付款金額」都是以「**總額**」為準。
- 審核者只對整張的總額按核准／不核准，不逐欄改金額（見上表核准金額說明）。

### 實際付款金額怎麼認列（已確認）
- 每張付款憑單提出後也要送審。
- 審到**最後一關**後，由**支出課（財務人員）**到「付款憑單管理頁」點該筆的**「確認付款」**。
- 一按「確認付款」，該張憑單轉為「已付款」，同時**認列為實際付款金額**（通常＝該憑單的核准金額）。

### 一個完整例子（母單 3000）
1. 資金分配申請 3000，最終核准 3000 → 資金分配列表「核准金額」顯示 3000。
2. 這張母單建立兩張付款憑單，各 1000、2000，匯給不同對象 → 付款憑單列表「核准金額」欄分別顯示 1000、2000。
3. 兩張都確實付款（財務各按「確認付款」）→ 「實際付款金額」欄分別認列 1000、2000，狀態變「已付款」。
4. 1000 + 2000 ＝ 3000，母單額度用完 → 母單結案。

## 五、確認狀態

本基準的核心邏輯已由 Yumin 於 2026-07-13 全數確認：

- **剩餘金額計算時機**：付款憑單**一建立（新單／審核中）就先預扣**佔用母單額度，不是等「確認付款」才扣；完整規則見《資金分配_剩餘金額計算邏輯規格》。
- **核准金額**：資金分配的核准金額**只能等於或下修**，不能上修；付款憑單只做核准／不核准、不編輯金額。

### 新增欄位／元素說明
- **付款分類**：財務作業用的付款方式／批次分類（觀察到的值：銀付、整批匯）。出現在審核管理頁與財務頁，一般列表頁沒有。
- **付款執行**：**財務頁專屬**。每一列一顆「確認付款」按鈕，按下即執行付款、把單轉「已付款」；可勾全選後「批次確認付款」。
- **帳戶分頁**：付款憑單依「出款帳戶」分成主要帳戶／營運資金／拆解機構…等分頁，各分頁標頭顯示該帳戶的「已核准總額」與「實際付款總額」（財務頁再多「已執行」）。
