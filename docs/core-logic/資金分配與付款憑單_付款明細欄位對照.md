# 資金分配申請單 × 付款憑單 — 付款明細欄位對照與金額關聯

> 目的：釐清兩張單據「付款明細」的欄位、各欄位意義、彼此金額關聯，以及舊系統（bci-financial.com）與新系統（allocation.boptaipei.com）的落差，作為調整 form-settings 前的確認依據。
> 依據：Yumin 提供的舊系統三張截圖（資金分配申請單 20260610015、其底下兩張付款憑單 …001 / …002，皆已付款）。

---

## 一、用同一筆實例貫穿理解（Fireflies.ai 6月平台費）

一張資金分配申請單 `20260610015`，核准金額 **2,864**，底下建了 **兩張**付款憑單，把 2,864 拆給兩個不同對象：

| 付款憑單 | 採購單號 | 付款對象（受款人） | 付款方式 | 總額 |
|---|---|---|---|---|
| 第一張 | 20260610015**001** | 黃鈺閔_E0023（申請人本人，代墊退款） | 銀行轉帳 | **2,298** |
| 第二張 | 20260610015**002** | 國稅局_S110（稅額） | 臨櫃繳款 | **566** |

2,298 + 566 = **2,864** ＝ 資金分配核准金額，剛好用完 → 剩餘金額 0。

## 二、資金分配申請單「付款明細」欄位 —— 新舊系統的關鍵差異（Yumin 補充）

### 舊系統：每項只有一個「金額」欄
| 欄位 | 意義 | 實例值 |
|---|---|---|
| 幣別 | 幣別 | 台幣 |
| 會計科目 | 會計科目代號 | 0000 D3選 |
| 摘要/用途說明 | 用途 | Fireflies.ai 6月平台費(4人) |
| **金額** | 整筆金額（單一欄） | **2,864** |
- 舊系統痛點：每項只有「金額」一欄，職員要**自己另外開頁面算稅額與手續費**，算完再回來手動填。
- 區塊右上「**合計付款金額**」＝所有項目金額加總，此數字即列表的「申請金額」。

### 新系統：改良為 費用／手續費／稅額（自動算稅，避免手算）
| 欄位 | 意義 | 圖三實例（項目1 / 項目2） |
|---|---|---|
| 單據種類 | 憑證類型 | 國內公司 |
| 費用項目（細項） | 費用細項 | 1.1 招募_廣告費 |
| 幣別 | 幣別 | 台幣 |
| 稅額選擇 | 選哪一種稅 → 驅動稅額自動計算 | 境外稅 |
| 摘要/用途說明 | 用途 | BOP活動費 茶水 / 餐點 |
| **費用** | 這項的費用（稅基） | 5000 / 1000 |
| **手續費** | 手續費 | 50 / 10 |
| **稅額** | **選了稅額選擇後自動算出並填入**（費用×稅率） | 1250 / 250 |
| （總額） | **費用＋手續費＋稅額，右上角自動彙總**（無獨立欄位） | 項目1=6,300、項目2=1,260 |

- 新系統設計意圖：職員只要填**費用＋手續費**、選**稅額選擇**，**稅額自動填入**，右上角「總額」自動加總（圖三：費用6,000＋手續費60＋稅額1,500＝**總額7,560**），就是這張資金分配要申請的金額，顯示在列表「申請金額」（圖四 my-funds 顯示 7,560）。
- 可**新增多個項目**（新增項目），總額為所有項目加總。
- **結論：資金分配的付款明細維持現狀正確**，不需要加可填的總額欄——它的設計本來就是自動算稅、自動加總，避免職員手算。

> **會計科目**：新系統**刻意先不顯示**（欄位項目會改、後台已建表但尚未匯入資料），資金分配與付款憑單兩張表單目前都不放會計科目欄，符合預期，不用調整。

## 三、付款憑單「付款明細」欄位（舊系統）

| 欄位 | 意義 | 第一張值 | 第二張值 |
|---|---|---|---|
| 幣別 | 幣別 | 台幣 | 台幣 |
| 會計科目 | 會計科目 | 0000 D3選 | 0000 D3選 |
| 摘要/用途說明 | 用途 | Fireflies.ai 6月平台費(4人) | Fireflies.ai 6月平台費(4人)-稅 |
| 未稅金額 | 稅前金額（選填，可留 0） | 0 | 0 |
| 稅額 | 稅額（選填，可留 0） | 0 | 0 |
| **總額** | **這張憑單實際要付給這個對象的金額（可直接填）** | **2,298** | **566** |

- 重點：**付款憑單的付款明細有三個金額欄「未稅金額／稅額／總額」**，比資金分配多了拆解。
- 實務上（本例）：申請人**直接填「總額」**（2,298、566），未稅金額與稅額都留 0 —— 代表「總額」是**可獨立手動填的欄位**，不是被未稅金額＋稅額鎖死自動算出來的。
- 未稅金額／稅額保留是給「需要拆稅務明細」的情境用（本例不需要就留 0）。

## 四、付款憑單最上方的「已核准資金分配申請單」對照區塊（舊系統，重點！）

每張付款憑單頁面最上方，都有一整條**父層資金分配申請單的摘要列**，欄位是：

`資金分配申請單號 ｜ 申請處別 ｜ 申請課別 ｜ 申請日期 ｜ 申請人 ｜ 職稱 ｜ 核准金額 ｜ 剩餘金額 ｜ 幣別 ｜ 機構 ｜ 出款帳戶 ｜ 費用項目 ｜ 項目`

實例值：`20260610015 ｜ 第七B處 技術研發中心 ｜ 第21B-1課 新技術研發部 ｜ 2026-06-10 ｜ 黃鈺閔 ｜ …課長 ｜ 核准金額 2,864 ｜ 剩餘金額 0 ｜ 台幣 ｜ BIZ ｜ 主要帳戶 ｜ 6.2 數位平台費用 ｜ Fireflies.ai 6月平台費(4人)`

- 意義：讓職員在建立/審核這張付款憑單時，**一眼看到母單核准了多少（2,864）、目前還剩多少可以分配（0）**，以及母單的處別/課別/費用項目/項目等完整背景。
- **這比我目前新系統做的對照卡片（只有核准金額／剩餘／本張金額三個數字）詳細很多**——舊系統是母單的一整條列（13 欄），對齊資金分配列表的欄位組合。
- 另外區塊右上還有「剩餘金額加總 / 最終剩餘總額」的即時試算（舊系統對已付款單顯示 0 與負數，屬其計算呈現方式；新系統我改用「剩餘(不含本張) / 送出後剩餘」語意更清楚，見第六節）。

## 五、三個金額層次的關聯（核心邏輯）

```
資金分配申請單
  核准金額 = 2,864（財務長核定，這是可用「額度」上限）
        │
        │  拆成多張付款憑單，各填「總額」
        ▼
  付款憑單1 總額 2,298（付黃鈺閔）
  付款憑單2 總額 566  （付國稅局）
        │
        ▼
  剩餘金額 = 核准金額 − Σ(各付款憑單總額)
          = 2,864 − 2,298 − 566 = 0  → 額度用完
```

- **資金分配的「金額/核准金額」** = 整筆額度總數（單一數字）。
- **付款憑單的「總額」** = 從這筆額度裡，分給某個付款對象的錢（可多張、加總不可超過核准金額）。
- **剩餘金額** = 核准金額 − 所有付款憑單總額加總；歸零即結案（已付款）。

## 六、新系統現況落差（要調整的地方）

### 落差 1：付款憑單「付款明細」沒有「總額」欄位
- 新系統付款憑單付款明細欄位是：**費用 ｜ 手續費 ｜ 稅額**（三欄），總額只在區塊右上角自動加總顯示（費用＋手續費＋稅額），**沒有一個「總額」格子可以填**。
- 舊系統是：**未稅金額 ｜ 稅額 ｜ 總額**，且「總額」可直接手動填（本例就是直接填 2,298 / 566）。
- 我已完成的程式：「總額欄位若存在，可手動輸入、改過就不被自動加總覆寫」——但表單設定裡根本沒建這個總額欄位（`taxConfig.totalFieldId` 是空的），所以畫面上沒有格子。
- **要做的**：在 form-settings 的付款憑單「付款明細」區塊**新增一個「總額」數字欄位**，並把稅額選擇的「總額欄位」設定指到它。（form-settings 頁本來就有這個能力，只是當初沒設。）

### 落差 2：欄位命名維持新系統（費用/手續費/稅額）＋加總額（已釐清，採此做法）
| | 舊系統付款憑單明細 | 新系統付款憑單明細（現況） | 調整後 |

| 金額欄 | 未稅金額 ／ 稅額 ／ **總額**(可填) | 費用 ／ 手續費 ／ 稅額（總額只在右上自動顯示，無欄位） | 費用 ／ 手續費 ／ 稅額 ／ **總額**(可填，預設自動加總、可手動覆寫) |

- 只在付款憑單付款明細**加一個「總額」欄位**：預設＝費用+手續費+稅額自動加總；使用者要直接填（如給本人 2,298、給國稅局 566）時可手動覆寫（程式已支援）。

## 七、我對「資金分配 vs 付款憑單」關聯的理解總結（請確認）

1. **資金分配 = 一筆額度**：付款明細只有單一「金額」＝總申請額（含所有費用），核准後成為可分批使用的額度上限。
2. **付款憑單 = 從額度撥給某對象的一筆錢**：付款明細有「總額」（可直接填），一張母單可拆多張、各給不同受款人，各張總額加總 ≤ 核准金額。
3. **未稅金額/稅額 是選填拆解**：需要時填，不需要就留 0、直接填總額（本例即如此）。
4. **剩餘金額 = 核准金額 − Σ付款憑單總額**，歸零即「已付款」結案。
5. **付款憑單頂部對照區塊 = 母單的完整摘要列**，讓人建立/審核時對照母單核准了多少、還剩多少。

以上理解若正確，接下來要做的就是：在 form-settings 付款憑單付款明細**補一個可填的「總額」欄位並掛上稅額設定的總額欄位**（落差 1 必做），落差 2、3 再看你要不要一起處理。
